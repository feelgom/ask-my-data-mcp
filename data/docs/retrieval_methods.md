# 검색 방식 비교

## BM25
BM25는 질의와 문서에 같은 단어가 등장하는 정도를 기반으로 점수를 계산하는 키워드 검색 방식입니다.
제품명, 오류 코드, 고유명사처럼 정확한 문자열 일치가 중요한 질의에 강하고 빠르게 실행할 수 있습니다.
반면 의미는 비슷하지만 표현이 다른 문서를 놓칠 수 있습니다.

## Dense Retrieval
Dense Retrieval은 질의와 문서를 임베딩 벡터로 변환한 뒤 의미적 유사도를 비교합니다.
동의어와 자연어 표현 차이에 강하지만, 숫자나 희귀 키워드의 정확한 일치에서는 BM25보다 약할 수 있습니다.
임베딩 생성과 벡터 검색 비용도 함께 측정해야 합니다.

## Hybrid Retrieval과 RRF
Hybrid Retrieval은 키워드 검색과 Dense Retrieval의 결과를 결합합니다.
RRF(Reciprocal Rank Fusion)는 각 검색기가 반환한 순위를 이용해 결과를 합치는 단순하고 강건한 방법입니다.
서로 다른 검색기의 점수 범위를 직접 정규화하지 않아도 되며, 어휘 일치와 의미 검색의 장점을 함께 활용합니다.

## 실험 선택 기준
정확도만으로 검색 방식을 고르지 않습니다. 언어, 문서 길이, 최신성, 지연 시간, 인덱스 비용을 함께 비교합니다.
작은 데이터셋에서는 BM25가 충분할 수 있고, 표현이 다양한 데이터에서는 Dense 또는 Hybrid 방식이 유리할 수 있습니다.
